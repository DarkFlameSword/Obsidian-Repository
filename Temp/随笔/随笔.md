# 新增逻辑
- 对于主界面的现有5个子界面“录入，推荐，白名单，强队，设置”，需要重新设计布局和名字，要求如下：
	- 对于原始“录入，推荐，白名单，强队”中，各个属性组合设置成可交互的按钮，点击或悬停2s后，出现悬浮框展示该组合下所有精灵的名字
	- “录入”改名为“实时权重录入”
	- “推荐”改名为“当前属性推荐”
	- 移除原始“设置”按钮，在主界面右下角新增一个齿轮按钮。点击齿轮之后显示当前“设置”子界面
	- 移除原始“白名单”按钮，在“设置”子界面新增一个“双属性名单录入”按钮，点击“双属性名单录入”按钮后进入原始“白名单”子界面
	- “强队”改名为“主流配队录入”

# 业务需求
- 主界面”主流配队录入“的子界面按钮应该在”当前属性推荐的前面
- 对于所有包含属性组合的界面（实时权重录入，主流配队录入，双属性名单录入），属性组合应该有合理的排序（举例：先单属性，后双属性；如果单属性顺序是“通用，草，水，火”，那么双属性顺序应该是“通用草，通用水，通用火）
- 对于”实时权重录入，主流配队录入“，在选中某个组合的属性后，这个属性的组合应该有”已选中“的视觉提示
- 对于”实时权重录入，主流配队录入“，在点击某个组合的属性后，应该是让用户点击该属性组合子菜单下的精灵名字，然后录入界面显示：该属性：精灵名字
