## 1. 确认本地 B 项目连接的github仓库地址
进入 B 项目目录：
```
cd /path/to/B git remote -v
```
看输出是不是 B 的 GitHub 仓库，比如：
```
origin  git@github.com:YourName/B.git (fetch) origin  git@github.com:YourName/B.git (push)
```
如果这里还是 A 的仓库地址，那就是没改过来。  
```
git remote set-url origin git@github.com:YourName/B.git
```

## 2. 检查 SSH Key 使用情况

如果 **两个项目在不同 GitHub 账号下**，为两个Github账号生成各自的SSH

新建文件 `C:\Users\<你的用户名>\.ssh\config`（注意这个config文件没有后缀名）：
```
Host github-A # 这里的地址名可以自定义,但是后面remote远端仓库的时候也要使用这个名字
	HostName github.com   
	User git   
	IdentityFile ~/.ssh/A项目的.pub文件名字
	 
Host github-B   
	HostName github.com   
	User git   
	IdentityFile ~/.ssh/B项目的.pub文件名字
```

项目 A 用 `github-A`，项目 B 用 `github-B`，互不冲突. 
修改项目 B 的远程地址，让它用 `github-B` 这个别名：
`git remote set-url origin git@github-B:YourName/B.git`

## 3. 推送测试

在 B 目录下：

`git push origin main`

（如果你分支不是 `main`，替换成你的分支名）
