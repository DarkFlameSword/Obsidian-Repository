## 1. 确认本地 B 项目连接的github仓库地址
进入 B 项目目录：
```
cd /path/to/B git remote -v
```
看输出是不是 B 的 GitHub 仓库，比如：
```
origin  git@github.com:YourName/B.git (fetch) origin  git@github.com:YourName/B.git (push)
```
如果这里还是 A 的仓库地址，那就是没改过来。  
```
git remote set-url origin git@github.com:YourName/B.git
```

## 2. 检查 SSH Key 使用情况

很多人遇到的问题是 **两个项目在不同 GitHub 账号下**，而本地只配置了一个 SSH Key。  
检查当前用哪个 key 连接：

`ssh -T git@github.com`

如果返回的是 A 账号的信息，但你要推送到 B 的仓库（属于另一个账号），就会推送失败。

解决办法：

1. 生成一个新 key（如果 B 的账号需要新的）：
    
    `ssh-keygen -t ed25519 -C "your_email@example.com"`
    
    保存成 `~/.ssh/id_ed25519_B`
    
2. 编辑 `~/.ssh/config` 文件（没有就新建）：
    
    `Host github-A   HostName github.com   User git   IdentityFile ~/.ssh/id_ed25519_A  Host github-B   HostName github.com   User git   IdentityFile ~/.ssh/id_ed25519_B`
    
3. 修改项目 B 的远程地址，让它用 `github-B` 这个别名：
    
    `git remote set-url origin git@github-B:YourName/B.git`
    

这样，项目 A 用 `github-A`，项目 B 用 `github-B`，互不冲突。

## 3. 推送测试

在 B 目录下：

`git push origin main`

（如果你分支不是 `main`，替换成你的分支名）
