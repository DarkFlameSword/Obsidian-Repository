## 🖥️ CLI 启动命令

| 命令                                    | 说明                      |
| ------------------------------------- | ----------------------- |
| `claude`                              | 启动交互式会话                 |
| `claude -c`                           | 恢复最近一次会话                |
| `claude -r "名称"`                      | 按名称恢复指定会话               |
| `claude -p "提示词"`                     | 非交互模式，发送后直接退出（适合脚本/CI）  |
| claude --dangerously-skip-permissions | 安全隐患命令                  |
| claude --enable-auto-mode             | 在claude自带的5层安全评估下自动处理许可 |

## 💬 会话管理（Slash 命令）

在交互模式中，输入 `/` 可查看所有可用命令，继续输入字母可筛选。

| 命令                      | 说明                                |
| ----------------------- | --------------------------------- |
| `/compact`              | 压缩对话历史，释放上下文空间（建议上下文用量超过 80% 时使用） |
| `/clear`                | 完全清空上下文（切换到全新任务时使用）               |
| `/resume` 或 `/continue` | 恢复之前的会话，无参数则打开选择器                 |
| `/fork`                 | 从当前状态创建对话分支，可探索不同方案               |
| `/rename`               | 给当前会话命名，方便以后查找                    |
| `/rewind`               | 回退对话及代码到之前状态（按 Esc+Esc 触发菜单）      |

2026 年更新后，`/rewind` 支持仅回退代码或仅回退对话两种模式，在实验性重构时非常实用。

## ⚙️ 模型与模式

| 命令              | 说明                              |
| --------------- | ------------------------------- |
| `/model sonnet` | 切换到 Sonnet 模型（日常任务，性价比高）        |
| `/model opus`   | 切换到 Opus 模型（复杂架构、疑难 bug）        |
| `/fast`         | 切换快速模式（速度提升 2.5x，成本更高）          |
| `/plan`         | 进入规划模式，先制定方案再执行                 |
| /model opusplan | 复杂推理时使用opus4.6，然后切换到sonnet4.6执行 |

## 🛠️ 内置技能（Skills）

技能（Skills）是基于提示词的能力，调用后会将指令加载到上下文，可执行复杂的多步骤工作流。 

|命令|说明|
|---|---|
|`/review`|代码审查|
|`/debug`|调试辅助|
|`/simplify`|简化/优化代码（建议在 AI 生成代码后运行）|
|`/batch`|批量并行执行重复性任务（如批量修改 API 端点）|
|`/loop`|循环迭代任务|

## ⌨️ 快捷键

按下 Esc+Esc 打开回退菜单，支持选择性地仅回退代码或仅回退对话；Alt+P 可在不清空输入框的情况下切换模型；Alt+T 切换深度推理（扩展思考）模式。 

|快捷键|说明|
|---|---|
|`Esc + Esc`|打开回退（Rewind）菜单|
|`Alt + P`|切换模型|
|`Alt + T`|开关扩展思考模式|
|`Shift + Tab`|切换自动接受权限确认模式|

## 🐚 Shell 直通

使用 `!` 前缀可直接执行 shell 命令，输出结果会自动加入对话上下文，无需 Claude 审批。 
bash

```bash
! npm test          # 运行测试，结果加入上下文
! git log --oneline -5   # 查看最近提交
! cat .env.example  # 查看文件内容
```

## 📁 自定义命令/技能

可以创建自定义 slash 命令，作为常用提示词的快捷方式：

bash

```bash
# 项目级命令
mkdir -p .claude/commands
echo "分析此代码的性能问题并提出优化建议：" > .claude/commands/optimize.md

# 全局命令（所有项目可用）
mkdir -p ~/.claude/commands
echo "检查此代码的安全漏洞：" > ~/.claude/commands/security.md
```

---

输入 `/help` 可在 Claude Code 内随时查看完整命令列表
