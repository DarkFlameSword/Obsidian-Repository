双线性插值要解决一个具体问题：
当我们用单应性矩阵的逆 $H^{-1}$ 把输出像素映射回源图像时，得到的坐标 $(x, y)$ 往往不是整数——比如落在 $(3.7,\ 2.4)$ 这样的位置，但图像只在整数格点上有像素值，怎么办？

三种插值方法的选择：最近邻（直接取最近格点，快但锯齿）、双线性（用周围 4 个点加权，平滑）、双三次（用 16 个点，更锐利但更慢）。图像拼接通常用双线性，在质量和速度之间取得最好平衡。

---

## 核心公式：先沿 x 插值，再沿 y 插值

设非整数坐标落在 $(x, y)$，令 $i = \lfloor x \rfloor$，$j = \lfloor y \rfloor$，小数部分 $\alpha = x - i$，$\beta = y - j$：

$$f(x,y) = (1-\alpha)(1-\beta),f_{i,j} + \alpha(1-\beta),f_{i+1,j} + (1-\alpha)\beta,f_{i,j+1} + \alpha\beta,f_{i+1,j+1}$$

每个角落的权重 = **水平方向距离 × 垂直方向距离**（到对角的距离，不是到自身的距离）。

下面的交互演示让你拖动采样点，实时看到四个权重如何随位置变化：右边的矩形图是理解双线性插值最直观的方式：**每个角点的权重 = 采样点到对角所围成的矩形面积**。当采样点靠近哪个角，那个角的权重就越大——这正是"距离越近贡献越大"的几何体现。拖动采样点，四块面积此消彼长，始终加起来等于 1。

## 为什么是"双线性"

"双线性"的含义是：先在一个方向线性插值，再在另一个方向线性插值。两步都是线性的，合起来对坐标是双线性的（不是二次曲线）：

$$f(x,y) = \underbrace{(1-\beta)}_{\text{上行权重}} \underbrace{\Big[(1-\alpha),f_{i,j} + \alpha,f_{i+1,j}\Big]}_{\text{第一步：沿 x 插值，得 } f_{\text{top}}} + \underbrace{\beta}_{\text{下行权重}} \underbrace{\Big[(1-\alpha),f_{i,j+1} + \alpha,f_{i+1,j+1}\Big]}_{\text{第二步：沿 x 插值，得 } f_{\text{bot}}}$$

先固定 $y$，在上边和下边各做一次线性插值；再对两个结果做一次线性插值。两步顺序可以互换，结果相同。

## 连接回图像拼接

在 `cv2.warpPerspective` 内部，整个流程是这样的：

$$\text{对输出图每个像素 } (x', y') \text{：} \quad \begin{bmatrix} wx \ wy \ w \end{bmatrix} = H^{-1} \begin{bmatrix} x' \ y' \ 1 \end{bmatrix} \quad\Rightarrow\quad (x, y) = \left(\frac{wx}{w},\ \frac{wy}{w}\right)$$

得到非整数坐标 $(x, y)$ 后，提取 $i = \lfloor x \rfloor$，$j = \lfloor y \rfloor$，$\alpha = x - i$，$\beta = y - j$，读取源图四个邻居，按权重 $(1-\alpha)(1-\beta)$，$\alpha(1-\beta)$，$(1-\alpha)\beta$，$\alpha\beta$ 加权求和，写入输出像素。

整个过程对输出图的**每个像素执行一次**，1000×1000 的图就是 100 万次这样的计算——这也是为什么 warping 在 GPU 上做比 CPU 快几十倍，因为每个像素的计算完全独立，天然适合并行。
